### 12. What was the most challenging task you faced in a project?

In one project, we had a performance bottleneck where API response times were exceeding 5 seconds for complex queries. I analyzed the issue using profiling tools, identified N+1 query problems, added proper indexing, implemented caching with Redis, and optimized the Hibernate fetch strategy. This brought the response time down to under 500ms — a 10x improvement.

### 13. How do you handle production issues or bugs?

I follow a structured approach:
1. **Reproduce** the issue in a lower environment if possible
2. **Analyze logs** using tools like ELK Stack or Splunk
3. **Identify root cause** through debugging and code review
4. **Fix and test** thoroughly with unit and regression tests
5. **Deploy** the fix following the proper release process
6. **Document** the RCA (Root Cause Analysis) for team awareness

---

### 14. Have you ever missed a deadline? How did you handle it?

In one instance, a requirement change came in mid-sprint which added unexpected complexity. I immediately communicated the risk to my lead, re-prioritized tasks, and negotiated a slightly extended timeline for the non-critical features. We delivered the core functionality on time and completed the rest in the next sprint. The key lesson was early communication.
