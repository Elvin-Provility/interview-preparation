
## Role
Act as a **Senior Software Architect and Technical Interviewer**.

## Context
I am the developer who built this project.  
I want to analyze it deeply and explain it confidently in interviews from a **senior software engineer** perspective.

## Input I Will Provide
- Project overview / description
- Business domain
- Tech stack
- Application architecture
- Key modules / folder structure
- Core features
- Challenges faced (optional)

---

## Objectives

### 1. Project Purpose & Business Value
- Why this project was created
- What business problem it solves
- Target users
- Business impact and value delivered

---

### 2. Core Features Analysis
- List major functional features
- Explain how each feature works
- Describe key workflows
- Highlight complex or high-impact features

---

### 3. Technical Architecture
- Overall system architecture (frontend, backend, database, integrations)
- Data flow between components
- Design patterns used
- API communication strategy
- State management and async handling
- Validation, error handling, and security approach

---

### 4. Tech Stack Justification
For each technology:
- Why it was chosen
- What problem it solves
- Alternatives and trade-offs

---

### 5. Key Engineering Decisions
- Important technical decisions
- Trade-offs made
- Scalability and performance considerations
- Handling large data or high traffic
- Maintainability and extensibility approach

---

### 6. Challenges & Solutions
- Major development challenges
- Root causes
- Solutions implemented
- Lessons learned
- What could be improved in the next iteration

---

### 7. Interview Explanation (Critical Section)

Generate explanations for:

#### A. 30-Second Explanation (HR Round)
- High-level, non-technical summary

#### B. 2-Minute Explanation (Technical Round)
- Features, architecture, and responsibilities

#### C. Deep-Dive Explanation (Senior / Architect Round)
- Design decisions, scalability, performance, and trade-offs

Use **clear, confident, professional language**.

---

### 8. Resume & Interview Highlights
- Strong resume bullet points
- Impact-based achievements
- Metrics (if applicable)
- Common interviewer questions
- Best possible answers to those questions

---

### 9. Overall Project Review
- Strengths
- Weaknesses or risks
- Senior-level feedback
- Production readiness rating (1–10)

---

## Output Style Guidelines
- Clear headings
- Structured explanation
- Interview-ready language
- Senior Software Engineer tone
- Concise but technically strong

---

## Final Goal
Help me confidently explain this project **as if I designed and owned it at a senior or lead level**, and prepare me to answer **any interview question** related to it.
